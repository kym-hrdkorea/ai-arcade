## 측정 요약

모바일 기준 뷰포트는 390x844로 측정했다.

- 대기 화면: `scrollHeight 868`, viewport `844`
  - 1페이지 목표보다 약간 길다.
  - 게임 전인데 드로잉 캔버스가 크게 노출되어 실제 행동과 맞지 않는다.
- 정답자 라운드: `scrollHeight 868`, viewport `844`
  - 캔버스와 정답 입력은 첫 화면 안에 들어온다.
  - 방 코드, 허브, 서버 상태, 나가기 버튼이 상단 150px 이상을 차지한다.
- 출제자 라운드: `scrollHeight 957`, viewport `844`
  - 제시어가 캔버스 아래로 밀려 첫 화면에서 그림과 동시에 보기 어렵다.
  - 도구 팔레트가 캔버스보다 먼저 크게 표시되어 실제 그림 영역이 늦게 나온다.

## 주요 발견

### P0. 모바일 출제자는 제시어와 캔버스를 동시에 봐야 한다

- 현재 출제자 화면은 도구 팔레트가 먼저 나오고, 캔버스 아래에 제시어가 있다.
- 익숙하지 않은 사용자는 “무엇을 그려야 하는지” 확인하려고 스크롤해야 한다.
- 요구사항:
  - 출제자에게 제시어를 캔버스 위 sticky chip 또는 캔버스 상단 고정 바에 표시한다.
  - 제시어는 큰 카드가 아니라 1줄 핵심 정보로 유지한다.
  - 모바일 출제자 첫 화면 안에 `라운드/시간/제시어/캔버스 상단/기본 도구`가 모두 보여야 한다.

### P0. 대기 참가자 화면에서 캔버스는 필요 없다

- 게임 시작 전 참가자는 그리거나 맞히지 않는다.
- 현재 대기 화면은 캔버스가 먼저 보여 설정/시작 상태와 참가자 확인이 아래로 밀린다.
- 요구사항:
  - 참가자 대기 화면은 “방 코드, 내 이름, 참가자 목록, 호스트 대기 안내” 중심의 lobby view로 분리한다.
  - 대기 중 캔버스는 숨긴다.
  - 모바일 대기 화면은 `scrollHeight <= viewportHeight`를 목표로 한다.

### P1. 플레이 중 방 코드는 상단에서 작아져야 한다

- 방 코드는 입장 전/대기 중에는 중요하지만, 라운드 중 플레이어에게는 거의 필요 없다.
- 현재 모바일 라운드 중에도 room code가 크게 남아 플레이 영역을 잡아먹는다.
- 요구사항:
  - 라운드 중 모바일에서는 room code를 1줄 작은 badge로 축소한다.
  - `허브로`, `서버 연결됨`, `나가기`는 overflow menu 또는 compact icon row로 접는다.
  - 실수 방지를 위해 `나가기`는 즉시 실행보다 확인 modal을 사용한다.

### P1. 정답자는 “관전”이 아니라 “정답자”로 보여야 한다

- 현재 그리지 않는 플레이어의 보드 badge가 `관전`으로 표시된다.
- 참가자는 자신이 게임에 참여 중인지 헷갈릴 수 있다.
- 요구사항:
  - 정답 입력 가능한 사람은 `정답자` 또는 `맞히는 중`으로 표시한다.
  - 실제 watcher 화면만 `관전`을 사용한다.

### P1. 드로잉 도구는 모바일에서 2단계 disclosure가 필요하다

- 현재 모바일 출제자 화면은 펜, 지우개, 굵기 3개, 색상 9개, 전체 지우기가 모두 펼쳐져 있다.
- 익숙한 사용자는 빠르게 색을 바꾸고 싶지만, 익숙하지 않은 사용자는 캔버스 접근이 늦어진다.
- 요구사항:
  - 1차 도구: 펜/지우개, 현재 색상, 현재 굵기, 전체 지우기.
  - 2차 도구: 색상 전체, 굵기 세부값은 접힘 팔레트 또는 bottom sheet.
  - 최근 사용 색상 3개 또는 추천 색상 4개를 우선 표시한다.

### P1. 정답 입력은 키보드가 올라와도 제출 버튼이 보여야 한다

- 현재 정답 입력과 제출 버튼은 첫 화면 안에 있으나, 모바일 가상 키보드가 올라오면 submit button이 가려질 가능성이 있다.
- 요구사항:
  - 입력 focus 시 답변 form을 viewport 하단 safe area 위에 sticky 고정한다.
  - Enter/Done으로 제출 가능해야 한다.
  - 제출 후 `제출 완료` 상태와 내 답변을 짧게 보여준다.

### P1. 대형 스크린은 시작 후 room code 노출 정책이 필요하다

- 대기 화면에서는 QR과 room code가 매우 잘 보인다.
- 라운드 시작 후에는 room code가 작게 남아 있다.
- 사내 워크숍은 괜찮지만 외부/공개 송출이면 late join과 방해 참가 위험이 있다.
- 요구사항:
  - 호스트 설정에 `시작 후 참가 코드 숨기기` 토글을 둔다.
  - 기본값은 사내 워크숍이면 표시, 공개 송출이면 숨김 preset을 제공한다.

### P2. 큰 화면 캔버스는 더 크게 써도 된다

- 대형 스크린 진행 중 화면은 오른쪽 참가자/점수 패널이 읽기 좋지만, 캔버스가 전체 화면 대비 조금 보수적이다.
- 요구사항:
  - 16:9 스크린에서는 캔버스가 가로폭 60-70%를 차지하도록 유지한다.
  - 참가자/점수 패널은 4명 이하일 때 더 압축하고, 캔버스 높이를 늘린다.

### P2. 초보자용 한 줄 안내가 더 구체적이어야 한다

- “호스트가 그리고 있습니다”는 상태는 알려주지만 다음 행동을 덜 알려준다.
- 요구사항:
  - 정답자: “그림을 보고 떠오르는 답을 아래에 입력하세요.”
  - 출제자: “제시어를 직접 쓰지 말고 그림으로 표현하세요.”
  - 대기자: “호스트가 시작하면 자동으로 게임 화면으로 이동합니다.”

## 플레이어 유형별 요구

### 모바일 플레이에 익숙하지 않은 사람

- 첫 화면에 할 일 하나만 보여야 한다.
- room code, 서버 상태, 라운드 설정 같은 운영 정보는 접혀 있어야 한다.
- 버튼 문구는 동사 중심이어야 한다: `입장하기`, `답 제출`, `그리기 시작`.
- `나가기`, `전체 지우기` 같은 위험 버튼은 확인 절차가 있어야 한다.

### 모바일 게임에 익숙한 사람

- 정답 입력까지 손가락 이동이 짧아야 한다.
- 제출 후 바로 다음 행동 상태를 알아야 한다.
- 색상/굵기 변경은 빠르되 캔버스 면적을 희생하지 않아야 한다.
- 라운드 전환 애니메이션은 짧고 상태 변화가 즉시 보여야 한다.

### 출제자

- 제시어, 타이머, 캔버스가 동시에 보여야 한다.
- 도구는 손가락이 닿기 쉬운 위치에 있어야 하지만 캔버스를 밀어내면 안 된다.
- 실수로 전체 지우기를 누르면 복구 또는 확인이 필요하다.

### 정답자

- 그림과 입력창이 한 화면 안에 있어야 한다.
- 정답 제출 가능 여부와 제출 완료 여부가 명확해야 한다.
- 다른 사람의 추측은 스포일러/혼란이 될 수 있으므로 라운드 중에는 최소화한다.

### 진행자

- 대기 중 참가자 수, 연결 상태, QR, 시작 버튼이 한 화면에 있어야 한다.
- 시작 후에는 스킵/리셋 같은 위험 제어가 보이되 실수 방지가 있어야 한다.
- 대형 스크린 URL과 참가 URL은 복사하기 쉬워야 한다.

### 대형 스크린 관람자

- 3-5m 거리에서 라운드, 남은 시간, 현재 출제자, 점수, 정답 공개 상태를 읽을 수 있어야 한다.
- 대기 중에는 QR과 코드가 충분히 커야 한다.
- 라운드 중에는 QR보다 그림과 결과가 우선이어야 한다.

## 개선 백로그

### Critical

1. 모바일 출제자 화면에서 제시어를 캔버스 위 고정 chip으로 이동.
2. 모바일 대기 참가자 화면에서 캔버스 제거, lobby summary로 대체.
3. 라운드 중 모바일 헤더 compact mode 적용.

### High

1. 정답자 role badge를 `관전`에서 `정답자`로 변경.
2. 드로잉 도구 모바일 compact 팔레트 설계.
3. 입력 focus 시 sticky answer bar와 Enter 제출 지원.
4. 위험 동작 `나가기`, `전체 지우기`, `방 리셋` 확인 절차 추가.

### Medium

1. 시작 후 대형 스크린 room code 숨김 설정 추가.
2. 초보자용 한 줄 행동 안내 개선.
3. 결과 화면에서 “내 점수/팀 점수/AI 결과” 정보 우선순위 재정리.
4. 화면 크기별 캔버스 목표 비율을 명시하고 회귀 테스트에 반영.

## 권장 수용 기준

- 모바일 390x844 기준:
  - 대기 화면 `scrollHeight <= 844`.
  - 정답자 라운드 화면에서 캔버스와 답변 form이 첫 화면 안에 들어온다.
  - 출제자 라운드 화면에서 제시어와 캔버스 상단이 첫 화면 안에 들어온다.
- 터치 타겟:
  - 주요 버튼 44px 이상.
  - 위험 버튼은 확인 또는 undo 제공.
- 대형 스크린:
  - 대기 중 QR/room code가 3m 거리에서 식별 가능.
  - 라운드 중 그림이 최우선이고 room code/운영 정보는 보조 정보다.
